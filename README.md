# 🌌 星空茶會 Starry Tea Party

多人即時線上飲料揪團點餐系統。本專案為 Java (Spring Boot) + H2 Database 後端，結合 Vanilla HTML / CSS / JS 前端的輕量且現代化 Web 應用程式。支援多人跨分頁、跨裝置即時同步，並具備豐富的防呆與安全防禦設計。

---

## 🚀 系統核心功能與現有進度

1. **探索店家與發起揪團**：
   * 大廳提供合作店家列表與基本資訊（如評分、距離等）。
   * 飲料外送低消門檻**統一修改為 200 元**。
   * **發起即開團流程**：主揪發起後房間狀態直接為 `active`（已開團/點餐中），直接開放並顯示複製跟團分享連結，免去繁瑣的手動開團步驟。

2. **跟團者自動加入與身分隔離**：
   * 跟團者透過分享連結進入後，系統會自動分配隨機匿名與獨立 `user_id` 靜默加入房間。
   * 使用 **`sessionStorage`** 保存身分，實現**「同瀏覽器、不同分頁」相互獨立**。方便在單台電腦開啟多個 Tab 模擬多人（跟團者 A、B、C）點餐。
   * 權限隔離：跟團者只能點餐並修改/刪除自己購物車的品項；只有主揪能在大總覽進行全局管理。

3. **外送運費階梯式計算**：
   * 運費依據店家距離（`distance`）階梯式計算：
     * 2 公里內：20 元
     * 2 到 3 公里：30 元
     * 超過 3 公里：每加 1 公里多 5 元。

4. **未達低消自動補差額機制**：
   * 若商品小計未滿 $200 低消門檻，大總覽（主揪與跟團者端）會自動顯示「未達低消差額補貼」列，並自動補足差額至實付總計中，主揪可流暢直接送出訂單給店家，防止被店家退單。

5. **後台狀態自動流轉**：
   * 訂單送出後，後台排程器（Scheduler）每 15 秒會自動在背景推進狀態：`processing` (製作中) $\rightarrow$ `delivering` (外送中) $\rightarrow$ `completed` (已完成)，無須人工點擊。

6. **商品限量庫存扣除閉環**：
   * 針對限量商品（如楊枝甘露限 5 杯），在點餐加入購物車時會安全扣除庫存，若刪除商品或解鎖修改則自動返還。

## ⚡ 踩雷紀錄與開發細節 (Bug Fixes)

在專案開發中，我們排除並優化了以下關鍵問題：

### 1. ⚠️ 跟團頁面 SyntaxError 崩潰
* **問題**：跟團者進入 `join.html` 後，頁面一片空白，無法載入選單與購物車。
* **原因**：`join.js` 中的大總覽統計函數 `renderTotalSummary()` 尾端多出了一個右花括號 `}`，導致瀏覽器語法解析失敗，整份 JS 拒絕執行。
* **解決**：移除多餘的 `}` 括號後恢復正常。

### 2. 👥 LocalStorage 導致的身分覆蓋衝突
* **問題**：在同一台電腦測試主揪與加團者時，購物車數據互相覆蓋，刪除別人的飲料會影響自己。
* **原因**：原本使用 `localStorage`，同瀏覽器分頁間資料共享，導致 `user_id` 互相覆蓋。
* **解決**：將前台所有使用者識別碼與暱稱的存取全面改為 **`sessionStorage`**。現在在同一個瀏覽器開多個分頁測試，每個分頁都會被識別為獨立的跟團者，互不干涉。

### 3. 🔒 越權訪問防禦 (BOLA / IDOR)
* **問題**：跟團者可透過修改前端代碼或 API 工具存取、刪除其他人的購物車資料。
* **原因**：後端未進行嚴格的權限驗證。
* **解決**：後端實作 `X-User-Id` Header 權限查驗。
  * 跟團者只能讀取/修改自己的購物車。
  * 大總覽數據僅在房間 `status !== 'active'`（結單後）才對跟團者開放。
  * 只有房間發起人（主揪）擁有全局管理與成員購物車代操作（提交/解鎖/刪除）權限。

## 🔮 未來展望與擴充方向

如果您未來想繼續完善這個專案，以下是極具實用價值的發展方向：

1. **💰 運費/低消差額自動平攤計算器**：
   * **痛點**：收錢時，主揪需要手動計算每個人要分攤多少運費與低消差額。
   * **方向**：在結單時，大總覽自動根據「已提交購物車的人數」將運費與差額平攤。在跟團者的頁面上顯示：「您本次應付：$X (飲料) + $Y (分攤運費) = **$Z**」，並提供收款 QR Code。

2. **⚡ 即時 WebSockets / SSE 同步**：
   * **優化**：目前系統使用 3 秒一次的短輪詢（Polling）與後端同步。
   * **方向**：引入 WebSocket 或 Server-Sent Events (SSE)，當有人加入、提交購物車或修改狀態時，所有人端即時秒級更新，減少伺服器 API 負擔。

3. **📅 店家營業時間與商品 availability 限制**：
   * **方向**：在資料庫中為店家增加營業時間段設定，或對特定商品增加「已售完」切換開關，前端菜單即時呈現售罄狀態且不允許點餐。

4. **💳 金流支付 API 整合**：
   * **方向**：串接綠界科技（ECPay）或 LINE Pay API，讓跟團者直接在跟團網頁完成付款，主揪不需再手動點名收錢。

## 🛠️ 本地啟動與開發指引

1. **運行後端**：
   ```bash
   mvn spring-boot:run
   ```
2. **訪問系統**：
   * 主揪端 / 首頁：`http://localhost:8080/index.html`
   * 分享給局域網他人（同 Wi-Fi）：請用 `ipconfig` 取得您電腦的 IPv4 地址，讓他人透過 `http://<您的IP>:8080/join.html?groupId=G-xxxx` 訪問。
